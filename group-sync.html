One-time setup · ~10 minutes

Make the savings board shared for everyone

Right now each phone saves its own numbers. With group sync turned on, when Oscar logs $200 on his phone, Leah, Raul, Rosa, Lily, Mike, and Yolee see it on theirs too — instantly.

Step 1 — Create a free Firebase project

  1. Go to Firebase Console and sign in with Google.
  2. Click Add project → name it colima-trip → continue → create.
  3. Open the project → click the web icon </> → register app name colima-trip-site.
  4. Copy the firebaseConfig object Firebase shows you.

Step 2 — Enable Firestore

  1. In Firebase, open Build → Firestore Database.
  2. Click Create database → start in production mode.
  3. Pick a region close to you (e.g. us-west) → enable.
  4. Open the Rules tab and paste:
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /trips/{tripId} {
      allow read, write: if true;
    }
  }
}

This is a family planning board on a private link — not a bank. Do not store account numbers here.

Step 3 — Paste keys into the site

  1. In this repo, open firebase-config.js.
  2. Set enabled: true.
  3. Paste your Firebase keys into the firebase section.
  4. Commit and push to main — GitHub Pages redeploys in ~30 seconds.

Step 4 — Tell the group

Everyone opens the same link: the trip site. Pick your name once — the site remembers it on your phone. Post contributions and everyone sees the live feed.

Need help? The status banner on the savings page turns green when sync is working. If it says “Sync issue,” double-check your Firebase keys and Firestore rules.
